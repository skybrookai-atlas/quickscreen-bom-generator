# QuickScreen BOM Calculation Rules

> Extracted from the official CTS order-form spreadsheets (V9.03 Side Frame, V9.1 Pedestrian Gate, V2.2 QSG Pedestrian Gates, V1.4 QSG Sliding Gates). These are the exact formulas the manufacturer uses.

---

## 1. Horizontal Slat Screen (QSHS Engine)

Source: `Order-Form-CTS-Slat-Side-Frame-Systems-V9.03`, sheet `QSHS`

### 1.1 Inputs (per panel row)
| Input | Description |
|-------|-------------|
| screen_type | FENCE, F-SECTION, or WINDOW |
| height_mm | Target screen height in mm |
| width_mm | Target screen/opening width in mm |
| f_section | Whether F-section fixing is used (true/false per side) |
| num_panels | Number of identical panels |
| slat_size | 65 or 90 (mm) |
| slat_spacing | Gap between slats: 5, 9, or 20 (mm) |
| colour | B, MN, G, SM, W, BS, D, M, P, PB, S |
| louvre_mode | true/false (if slats are angled as louvres) |

### 1.2 Width Deductions
The slat CUT LENGTH is the opening width minus a deduction that depends on screen type and louvre mode:

| Screen Type | Without Louvres | With Louvres |
|-------------|----------------|--------------|
| FENCE PANEL (overall opening size) | -15mm | -58mm |
| F-SECTION / VOID INFILL (overall opening size) | -15mm | -78mm |
| WINDOW SCREEN (edge-to-edge of side frames) | -6mm | -56mm |

```
slat_cut_length = width_mm - deduction
```

### 1.3 Slat Count (from Lookup Table)
The spreadsheet uses a lookup table mapping height → number of slats. The table is generated by:

```
# Height for N slats:
height_for_N_slats = ROUND(N × (slat_width + slat_spacing) - slat_spacing + 3, 0)

# Reverse: given target height, find N:
num_slats = VLOOKUP(height_mm, slat_table, 2, FALSE)
```

Where:
- `slat_width` = 65 or 90
- `slat_spacing` = 5, 9, or 20
- The +3mm accounts for the side frame lip

**Simplified programmatic formula:**
```python
def get_num_slats(target_height, slat_width, slat_spacing):
    """Find the number of slats that gives the closest height <= target"""
    n = 3  # minimum slats
    while True:
        h = round(n * (slat_width + slat_spacing) - slat_spacing + 3)
        if h > target_height:
            return n - 1
        n += 1

def get_actual_height(num_slats, slat_width, slat_spacing):
    """Calculate the actual achieved height for a given slat count"""
    return round(num_slats * (slat_width + slat_spacing) - slat_spacing + 3)
```

**Always report actual_height** — it almost certainly differs from the target. The user needs to know.

### 1.4 Centre Support Rails (CSR)
Width determines how many centre support rails are needed per panel:

| Panel Width (mm) | Centre Support Rails |
|-------------------|---------------------|
| 0 – 1999 | 0 |
| 2000 – 3999 | 1 |
| 4000 – 5999 | 2 |
| 6000+ | 3 |

```
num_csr_per_panel = VLOOKUP(width_mm, CsrWidths, 2)
total_csr = num_panels × num_csr_per_panel
```

**Exception:** If the user selects "No Centre Support Rails" (override), set to 0.

### 1.5 Cut Lengths
```
side_frame_cut_length = height_mm - 3     (mm)
centre_rail_cut_length = height_mm - 6    (mm)
```

### 1.6 Stock Length Optimisation
All extrusions are sold in stock lengths. The spreadsheet calculates how many pieces can be cut from one stock length:

```
slats_per_stock = ROUNDDOWN(stock_length / slat_cut_length, 0)
side_frames_per_stock = ROUNDDOWN(stock_length / side_frame_cut_length, 0)
csr_per_stock = ROUNDDOWN(stock_length / centre_rail_cut_length, 0)
```

**Stock lengths** (from product codes):
- Slats (XP-6100-S65 / QS-6100-S90): 6100mm
- Side Frame (QS-5800-SF): 5800mm
- CFC Cover (QS-5800-CFC): 5800mm
- Centre Support Rail (XP-5800-CSR): 5800mm
- F-Section (QS-5800-F): 5800mm

### 1.7 Stock Lengths to Order
```
slat_lengths_to_order = ROUNDUP(total_slats / slats_per_stock, 0)
side_frame_lengths_to_order = ROUNDUP(total_side_frames / sf_per_stock, 0)
cfc_lengths_to_order = side_frame_lengths_to_order   # always matches side frames
csr_lengths_to_order = ROUNDUP(total_csr / csr_per_stock, 0)
f_section_lengths_to_order = ROUNDUP(total_f_sections / f_per_stock, 0)
```

### 1.8 Accessories Calculation
```
# Side Frame Caps (sold as 2-packs = 1 pair top+bottom)
sf_cap_packs = total_side_frames / 2    # Each side frame gets 1 cap top + 1 bottom, sold as 2-pack

# Centre Support Rail Caps
csr_caps = total_csr    # 1 cap per centre rail

# Spacer Blocks (sold in 50-packs)
spacers_per_panel = 2 × (num_slats + 1)
total_spacers = SUM(spacers for all panels)
spacer_packs = ROUNDUP(total_spacers / 50, 0)

# Side Frame Screws (sold in 50-packs)
sf_screws = num_slats × 2 × 1.01    # 1% spare
csr_screws = num_slats × num_csr_per_panel
total_screws = sf_screws + csr_screws
screw_packs = ROUNDUP(total_screws / 50, 0)

# F-Section Fixing Screws (10G x 16mm colour-matched)
f_section_screws_per_panel = MAX(3, ROUNDUP(height_mm / 900 + 1, 0))
total_f_screws = total_f_sections × f_section_screws_per_panel

# Louvre Brackets (only if louvre mode)
louvre_brackets = num_slats × 2 × num_panels   # if louvre_mode = true
```

### 1.9 Product Code Pattern
```
{base}-{colour_suffix}
```
Examples:
- `XP-6100-S65-B` = 65mm slat, Black
- `QS-6100-S90-MN` = 90mm slat, Monument
- `QS-5800-SF-G` = Side Frame, Woodland Grey
- `QS-5800-CFC-SM` = CFC Cover, Surfmist
- `XP-5800-CSR-B` = Centre Support Rail, Black
- `QS-5800-F-B` = F-Section, Black
- `QS-SFCAP-Acc-2PK` = Side Frame Cap (nylon, colour neutral, 2-pack)
- `XP-CSRC-Acc` = Centre Support Rail Cap (nylon)
- `QS-SPACER-xxMM-50PK` = Spacers (xx = gap size: 05, 09, 20)
- `QS-SCREWS-50PK` = Screening Screws 50-pack
- `XP-SCREWS-Singles-Acc` = 10G x 16mm colour-matched F-section screws

## 2. Vertical Slat Screen (VS Engine)

Source: `Order-Form-CTS-Slat-Side-Frame-Systems-V9.03`, sheet `VS`

### 2.1 Key Differences from Horizontal
Vertical slat screens use the same extrusion products but oriented differently:
- **Slats run vertically** (cut to screen HEIGHT)
- **Side frames used as TOP and BOTTOM horizontal rails**
- **QS-5000-HORIZ U-channel** used as slat holder rail
- **QS-5800-F** used for the vertical slat extrusion

### 2.2 Inputs
| Input | Description |
|-------|-------------|
| slat_spacing | Gap between vertical slats (mm) |
| slat_size | 65 or 90 (width of vertical slat face) |
| panel_heights[] | Height of each panel zone |
| panel_widths[] | Width of each panel zone |
| num_panels | Number of identical panels per zone |

### 2.3 Key Formulas
The VS sheet uses a named range `VS_VertSlatMat` (vertical slat material table) for lookups.

**Authoritative vertical slat count formula** (from CTS - Vertical Screens tab):
```
num_slats = ROUNDDOWN((panel_width - 8 + slat_spacing) / (slat_spacing + slat_size), 0)
```
Where `8` = frame channel depth (4mm each side). Always use this — do NOT use the VLOOKUP approximation.

**Overhang / even distribution:**
```
overhang_each_side = ROUND((panel_width - 8 - num_slats × (slat_spacing + slat_size) + slat_spacing) / 2, 1)
```

```
# Slat cut length = panel height (vertical orientation)
slat_cut_length = panel_height

# Number of vertical slats per panel (use formula above, not VLOOKUP):
total_vertical_slats = VLOOKUP(panel_width, VS_VertSlatMat, 8, FALSE)  # legacy — prefer formula above

# Side frames become top/bottom rails (cut to panel width)
rail_cut_length = panel_width

# Stock optimisation (same ROUNDDOWN pattern):
slats_per_stock = ROUNDDOWN(stock_length / (slat_cut_length + Cut_Allowance), 0)
rails_per_stock = ROUNDDOWN(stock_length / (rail_cut_length + Cut_Allowance), 0)

# Stock to order:
slat_stocks = ROUNDUP(total_slats / slats_per_stock, 0)
rail_stocks = ROUNDUP(total_rails / rails_per_stock, 0)
```

Where `Cut_Allowance` is a small waste/kerf factor (typically ~3mm).

### 2.4 VS Screws
```
# Rounded up to nearest 10, then to 50-packs
raw_screw_count = total_slat_count × 1.01   # 1% spare
rounded_screws = ROUNDUP(raw_screw_count, -1)  # round to nearest 10
screw_packs = ROUNDUP(rounded_screws / 50, 0)
```

## 3. XPress Horizontal Slat Pedestrian Gate (XPHS Engine)

Source: `CTS-Pedestrian-Gate-V9.1-Lever-and-Knob`, sheet `XPHS`

### 3.1 Inputs
| Input | Description |
|-------|-------------|
| gate_width_mm | Width of gate opening |
| gate_height_mm | Height of gate |
| slat_spacing | Gap: 5, 9, or 20mm |
| gate_side_frame_size | 50, 60, or 65mm |
| num_gates | Quantity of identical gates |
| hinge_type | L (left) or R (right) |

### 3.2 Gate Blade Count
Gate blades are looked up by height from the `Gate_Blades` table:
```
# Number of blades per gate:
blade_count = VLOOKUP(gate_height_mm, Gate_Blades, 2, TRUE)  # approximate match

# Blade CUT length:
blade_cut_length = VLOOKUP(gate_height_mm, Gate_Blades, 4, TRUE)
```

The Gate_Blades table uses the same slat-height formula as horizontal screens:
```
height_for_N_blades = ROUNDDOWN(2 × stile_clearance + (N-1) × blade_width + N × slat_spacing + 1 × end_clearance, 0)
```

### 3.3 Gate Side Frame Cut Length
Gate side frames include a 66.5mm allowance per joint:
```
gate_sf_pieces_per_stock = MAX(1, ROUNDDOWN(stock_length / (gate_sf_cut_length + 66.5), 0))
gate_stop_pieces_per_stock = MAX(1, ROUNDDOWN(stock_length / (gate_stop_cut_length + 66.5), 0))
```

### 3.4 Centre Rail (Diagonal Brace)
Gates have a diagonal centre rail for rigidity:
```
centre_rail_length = ROUND(SQRT(gate_width^2 + gate_height^2), 0)   # Pythagorean diagonal
centre_rails_per_stock = ROUNDDOWN(stock_length / (centre_rail_length + Cut_Allowance), 0)
```

### 3.5 Gate Stop Screws
```
gate_stop_screws = MAX(4, ROUNDUP((gate_stop_cut_length - 100) / 500 + 1, 0)) × num_gate_stops
```

### 3.6 Stock Lengths to Order (Gates)
Same ROUNDUP pattern as screens:
```
slat_stocks = ROUNDUP((top_slats + bottom_slats) / slats_per_stock, 0)
blade_stocks = ROUNDUP(total_blades / blades_per_stock, 0)
sf_stocks = ROUNDUP(total_gate_sf / sf_per_stock, 0)
hd_rail_stocks = ROUNDUP(total_hd_rails / hd_per_stock, 0)
centre_rail_stocks = ROUNDUP(total_centre_rails / cr_per_stock, 0)
gate_stop_stocks = ROUNDUP(total_stops / stops_per_stock, 0)
```

### 3.7 Gate Products (from spreadsheet stock data)
| Component | Product Code Pattern | Stock Length |
|-----------|---------------------|--------------|
| 65mm Slat | XP-6100-S65-{col} | 6100mm |
| Gate Blade (65mm) | XP-6100-GB65-{col} | 6100mm |
| HD Top/Bottom Rail | XP-6100-HD6545-{col} | 6100mm |
| Gate Side Frame | Varies by size (50/60/65mm) | ~5800mm |
| Gate Stop | XP-4200-GSTOP-{col} | 4200mm |
| Gate Insert | Varies | - |
| Centre Rail (diagonal) | XP-5800-CSR-{col} | 5800mm |

### 3.8 Gate Screws (XPHS)
```
# For 2400mm stock slats, slats are divided by 3:
if stock_length == 2400:
    slat_order = ROUNDUP(total_slats / 3, 0)
else:
    slat_order = total_slats

# Gate screws sold in 10-packs:
screw_packs = ROUNDUP(total_screws / 10, 0)
```

## 4. Lever/Knob Gate (LKG Engine)

Source: `CTS-Pedestrian-Gate-V9.1-Lever-and-Knob`, sheet `LKG`

### 4.1 Key Differences from Standard Gate
- Uses a lock box with lever handle or knob
- Requires `StockLatchHt` and `LatchHtSel` named values for latch positioning
- Gate side frame height: `ROUND(gate_height - 3, 0)`
- Latch stile length: `ROUNDDOWN(StockLatchHt - LatchHtSel + gate_sf_height, 0)`

## 5. QSG Horizontal Slat Pedestrian Gate (QSG_H Engine)

Source: `CTS-QSG-Pedestrian-Gates-V2.2`, sheet `QSG_H_Calcs`

### 5.1 Named Constants (QSG System)
```
Wrail_H = rail width for horizontal orientation (mm) — from Tables sheet
Wslat_H = slat width (65 or 90mm)
Hgfc = gate frame clearance height (mm)
Gap = slat spacing
Hmin = minimum gate height
Hmax = maximum gate height
```

### 5.2 Key Formulas
```
# Infill height (slat area inside gate frame):
infill_height = gate_height - 2 × Wrail_H

# Gate frame overall height:
gate_frame_height = ROUNDUP(gate_height - INT(Hgfc), 0)

# Number of slats in gate:
num_slats = ROUNDUP((gate_height - 2 × Wrail - Gap) / (Wslat + Gap), 0)

# Gate height for N slats (lookup table):
height_for_N = ROUND(2 × Wrail_H + N × (Wslat_H + gap) + end_clearance, 0)
```

### 5.3 QSG Gate Stock Products
| Component | Code Pattern | Stock Length |
|-----------|-------------|--------------|
| QSG 65mm Horizontal Rail | QSG-4800-RAIL65-{col} | 4800mm |
| QSG 90mm Horizontal Rail | QSG-4800-RAIL90-{col} | 4800mm |
| QSG Gate Side Frame | QSG-GATESF-{spacing}-{col} | varies |
| QSG Gate Blade | QSG-GB65-{col} | varies |
| QSG Screw Cover | QSG-SC-10PK | 10-pack |
| QSG Rail Screws | QSG-RS-10PK | 10-pack |
| QSG Joiner Block 65mm | QSG-JBLOCK-65-4PK | 4-pack |
| QSG Joiner Block 90mm | QSG-JBLOCK-90-4PK | 4-pack |
| QSG Gate Frame Cap | QSG-FTC-50 or QSG-FTC-65 | each |
| QSG Flat Top Cap | QSG-FTP-50 | 50mm post cap |

### 5.4 QSG Gate Pricing
QSG gates are priced per lineal metre of slat and per lineal metre of frame:
```
gate_price = (lineal_metres_slat × slat_rate_per_LM) + (lineal_metres_frame × frame_rate_per_LM)
```

From the spreadsheet:
```
# Slat cost code lookup (based on finish type and size):
if finish == "Col" and size == 65: code = "SC-10315"
if finish == "Mil" and size == 65: code = "SC-1881"
if finish == "Img" and size == 65: code = "SLAT6516CTL"
if finish == "Col" and size == 90: code = "SC-10705"
if finish == "Mil" and size == 90: code = "SC-1882"
if finish == "Img" and size == 90: code = "SLAT9016CTL"
```

### 5.5 QSG Vertical Slat Gate (QSG_V)
Uses the same frame structure as QSG_H but:
- Slats oriented vertically
- Slat cut length = gate height (minus rail clearances)
- Number of slats determined by gate width and slat spacing
- Additional centre support rail may be needed for wide gates

## 6. QSG Sliding Gate (HSSG / VSSG Engines)

Source: `Order-Form-QSG-Sliding-Gates-V1.4`, sheets `HSSG` and `VSSG1`

### 6.1 Sliding Gate Inputs
| Input | Description |
|-------|-------------|
| height_from_ground | Height from ground level (mm) — includes 31mm ground clearance |
| opening_width | Opening width to cover (mm) |
| slat_size | 65 or 90mm |
| slat_spacing | Gap between slats (mm) |
| slat_colour | Colour code |
| frame_colour | Frame colour code |
| qty_gates | Number of identical gates |

### 6.2 Gate Overall Dimensions
```
# Ground clearance = 31mm (based on 16mm steel track; 35mm for 20mm aluminium track)
gate_height_overall = height_from_ground + ground_clearance_adjustment
gate_width_overall = opening_width + overlap   # overlap for pocket side
```

### 6.3 Horizontal Slat Sliding Gate (HSSG)
```
# Slat count per gate:
num_slats = XLOOKUP(gate_height_overall, HSSG_Hoa["INT(Ht)"], HSSG_Hoa["# Blades"])

# Slat cut length:
slat_length = gate_width_overall - rail_deduction

# Clear height calculation:
clear_height = gate_height_overall - top_rail_height - bottom_rail_height - 20 - 1

# Centre support rails per gate:
num_csr = lookup based on gate width (same thresholds as screen CSR)
```

```
# Track must extend beyond opening:
track_length_needed = gate_width_overall × 2   # full pocket + opening

```

### 6.6 Vertical Slat Sliding Gate (VSSG)
Same frame components as HSSG but:
- Slats oriented vertically (cut to gate height)
- Stile length = gate_height_overall - top_rail - bottom_rail deductions
- Number of slats determined by gate width and spacing
- Additional infill pieces between rails

```
gate_price = (lineal_metres_slat × slat_rate) + (lineal_metres_perimeter × frame_rate)
# Discounts applied per O_group table
```

### 7.2 Post Selection Rules
| Scenario | Recommended Post | Why |
|----------|-----------------|-----|
| Screen up to 1800mm high | 50x50mm (XP-2400-FP) | Standard strength |
| Screen 1800-2400mm high | 65x65mm HD (XP-2400-65HD) | Increased rigidity |
| Screen over 2400mm | 65x65mm HD (XP-5800-65HD) | Needs longer stock, cut to length |
| Windy/exposed location | 65x65mm HD | Wind load resistance |
| Sliding gate posts | 65x65mm steel (XPSG-2700-ST65) | Must handle gate weight |
| Pedestrian gate posts | 65x65mm HD recommended | Gate forces require strength |
| Corner posts (90° turn) | Match adjacent panels | Same size as run posts |
| POSTA letterbox | 50x50mm at 600mm centres | Required spacing for letterbox |

### 7.3 Post Mounting Accessories (per post)

**Method A: Concreted into Ground**
- Post length = screen_height + 450mm minimum in-ground
- Materials: concrete or Kwikset grout
- No base plate, domical cover, or dress ring needed
- Best for new installations in soil/turf

**Method B: Base-Plated to Concrete Slab**
- 1× Base Plate Set (XP-BP-SET or XP-65BP-SET)
- 1× Domical Cover (XP-DC-2P or XP-65DC-2P)
- 4× Dynabolts or concrete anchors (not in price list — BYO)
- Post length = screen height only (sits on slab)

**Method C: Core-Drilled into Concrete**
- 1× Dress Ring (XP-DR or XP-65DR)
- Chemical anchor (SOUD-VE400 or SOUD-CA1400)
- Post length = screen_height + 150-200mm (depth of core hole)

## 8. Panel & Post Layout Calculations

### 8.1 Straight Fence Run
```python
max_panel_width = 2600  # mm typical, 2000 for windy areas

panel_width = total_run_length / number_of_panels  # even distribution

```

### 8.2 With 90° Turns
Each turn adds:
- 1 corner post (replaces what would be an end post)
- Panels on each side of turn are independent

```python
# Example: L-shaped fence with sides A and B
panels_A = ceil(length_A / max_panel_width)
panels_B = ceil(length_B / max_panel_width)
total_panels = panels_A + panels_B
total_posts = panels_A + 1 + panels_B  # corner post is shared
```

# Gate posts are ADDITIONAL (or replace fence end posts)

## 11. XPress Plus Premium (XPL) — Insert-Based System

Source: `Order-Form-CTS-Slat-Side-Frame-Systems-V9.03`, sheet `XPL`

> **CRITICAL CONSTRAINT: XPL Premium side frame system is 65mm slat ONLY. No 90mm option exists.**

### 11.1 How XPL Differs from QSHS

QSHS counts individual slats; XPL counts **insert blades** held in slotted sections. The blade sits in a slot (not just stacked), so the centre-to-centre calculation is different.

### 11.2 XPL Physical Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `Ht_EndCap` | 3 mm | Height consumed by end cap (top and bottom) |
| `Cut_Allowance` | 2 mm | Saw kerf / waste per cut |
| `Wslot` | 66.5 mm | Width of each insert blade slot |
| `Ss` | 2 mm | Side clearance between frame and first blade slot |
| `Gs` (9mm nominal) | 8 mm actual | Actual gap at "9mm" setting |
| `Gs` (20mm nominal) | 20 mm actual | Actual gap at "20mm" setting |
| `Blade_cc` (9mm) | 74.5 mm | Blade centre-to-centre = 8 + 66.5 |
| `Blade_cc` (20mm) | 86.5 mm | Blade centre-to-centre = 20 + 66.5 |
| `Trim_allowance` (9mm) | 6 mm | Trim allowance for 9mm inserts |
| `Trim_allowance` (20mm) | 4 mm | Trim allowance for 20mm inserts |

### 11.3 XPL Height → Blade Count

XPL uses the `XPL_Ht` lookup table: each row D blades gives height:
```
height_for_D_blades = ROUNDDOWN(2×Ht_EndCap + 2×Ss + (D-1)×Gs + D×Wslot, 0)
```

To find blade count for a target height:
```
num_blades = VLOOKUP(target_height, XPL_Ht, 2, 0)   # exact match required
```

### 11.4 Width Deductions (XPL)

Uses the same `SlatDeductions` table as QSHS but with a louvre column:
```
slat_cut_length = width_mm - VLOOKUP(screen_type, SlatDeductions, 2 + (louvre_mode), FALSE)
```

| Screen Type | Without Louvres | With Louvres |
|-------------|----------------|--------------|
| FENCE PANEL | -15mm | -58mm |
| F-SECTION/VOID INFILL | -15mm | -78mm |
| WINDOW SCREEN | -6mm | -56mm |

### 11.5 Side Frame Cut Length (XPL)

```
side_frame_cut_length = height_mm - 2 × Ht_EndCap   # = height_mm - 6
```

### 11.6 Centre Support Rail (XPL)

```
csr_cut_length = height_mm - Gs × 2 - 3
# = height_mm - (actual_gap × 2) - 3
# NOTE: Different from QSHS formula (height - 6)
```

### 11.7 Fixing Types (XPL)

XPL supports F-Section and U-Channel (QSHS only has F-Section):
```
fixing_qty = panels × 2        # 2 fixings per panel (left + right)
fixing_cut_length = height_mm  # full screen height
```

**Stock lengths to order:**
```
f_section_stocks = ROUNDUP(fixing_qty / ROUNDDOWN(SF_stock_length / fixing_cut_length, 0), 0)
u_channel_stocks = ROUNDUP(fixing_qty / ROUNDDOWN(UC_stock_length / fixing_cut_length, 0), 0)
```

### 11.8 F-Section Screws (XPL)

Same formula as QSHS:
```
f_screws = panels × 2 × ROUNDUP((height_mm - 30) / 900 + 1, 0)
```

### 11.9 Slat Stock Optimisation (XPL)

```
slats_per_stock = ROUNDDOWN(slat_stock_length / (slat_cut_length + Cut_Allowance), 0)
slat_stocks = ROUNDUP(total_slats / slats_per_stock, 0)
```

### 11.10 Insert Quantity

```
inserts_per_panel = INT(available_height / Blade_cc)
total_inserts = SUM(inserts for all panels)
```

### 11.11 Pricing (XPL)

XPL applies a 5% markup to all line totals:
```
line_total = ROUND(SUM(material_costs) × 1.05, 2)
```

Rates come from `XpMaterials` named range (per LM by material and colour group).

## 12. BAYG — Buy As You Go System

Source: `Order-Form-CTS-Slat-Side-Frame-Systems-V9.03`, sheet `BAYG`

### 12.1 What Is BAYG?

BAYG uses the same side frame extrusions as XPL Premium but sells the spacer/slat assembly differently — spacers are a separate purchasable item ("buy as you go"). Slat widths available: 65mm or 90mm.

### 12.2 Key Differences from QSHS/XPL

- Uses `BAYG_ScrHts` lookup table (height → slat count) — NOT the XPL_Ht table
- Has F-Channel fixing (not F-Section or U-Channel)
- Spacers are sold separately and calculated differently
- No F-section for FENCE type screens

### 12.3 Slat Count (BAYG)

```
num_slats = VLOOKUP(target_height, BAYG_ScrHts, 2, 0)   # exact match lookup
```

The BAYG_ScrHts table is different from both QSHS's table and XPL_Ht.

### 12.4 Side Frames (BAYG)

```
side_frame_qty = panels × 2
side_frame_cut_length = height_mm - 2 × Ht_EndCap   # = height_mm - 6
```

### 12.5 CSR (BAYG)

```
csr_cut_length = height_mm - 3   # different from XPL (height - Gs×2 - 3)
```

### 12.6 Spacers (BAYG-specific)

```
spacers_per_panel = 2 × (num_slats + 1)
# Spacers sold in bags — calculate bag quantity:
spacer_bag_qty = ROUNDUP(total_spacers / bag_size, 0)
```

**Important note (from spreadsheet):** At 20mm spacing, the snap-in spacer is physically **21mm high** (not 20mm) to align with the gate frame spacing. The ordering code still refers to "20mm" spacing.

### 12.7 F-Channel Fixing (BAYG)

F-Channel is NOT used for FENCE type screens — only for other screen types:
```
f_channel_qty = IF(screen_type.startsWith("FENCE"), 0, panels × 2)
f_channel_cut_length = height_mm   # full screen height
```

### 12.8 Screws (BAYG)

```
sf_screws = num_slats × 2 × 1.01   # 1% spare
csr_screws = num_csr × num_slats
total_screws = ROUNDUP((sf_screws + csr_screws) × 1.01, 0)  # overall 1% spare
```

### 13.3 Width Deductions for Louvre Mode

For XPL/BAYG: max achievable height is limited to **2000mm** in louvre mode (vs full range otherwise).
